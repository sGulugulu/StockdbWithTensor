## 新版正式实施计划同步

### 一、正式研究范围调整

1. 正式 A 股实验范围不再以中证 A500 为目标，而是以以下三类核心指数股票池为主：
   - 沪深300（`HS300`）
   - 上证50（`SZ50`）
   - 中证500（`ZZ500`）
2. 正式数据底座不再采用“三指数并集公司”作为长期主存储范围，而是改为：
   - 全 A 股主数据统一保存
   - `HS300`、`SZ50`、`ZZ500` 的成分股历史分别单独保存
   - 全 A 可交易股票池历史单独保存
3. 旧有样例数据、样例配置和样例输出继续保留，仅作为 smoke test、联调和演示输入，不与正式全量数据混用。

### 二、正式数据窗口与存储原则

1. 本阶段正式全量数据窗口固定为：
   - 起始日期：`2015-01-01`
   - 截止日期：`2026-04-01`
2. 财务数据与报告数据的抓取截止日期同样固定为 `2026-04-01`。
3. 正式数据先落盘为 `CSV`，便于检查、抽样验证和脚本兼容。
4. 在 `CSV` 稳定后，再生成对应的 `Parquet` 文件，用于后续高性能读取和大规模计算。
5. `CSV` 与 `Parquet` 的命名、字段合同、时间范围和 manifest 需要保持一一对应。

### 三、正式数据底座结构

1. 全 A 股行情数据统一保存为正式主数据，不按指数重复保存多套完整行情文件。
2. 全 A 股财务/报告数据统一保存，并按表类型分别落盘，至少包括：
   - `profit_data`
   - `operation_data`
   - `growth_data`
   - `balance_data`
   - `cash_flow_data`
   - `dupont_data`
   - `performance_express_report`
   - `forecast_report`
3. universe 层单独维护历史文件，不与主数据混写，至少包括：
   - `HS300` 成分股快照与变更记录
   - `SZ50` 成分股快照与变更记录
   - `ZZ500` 成分股快照与变更记录
   - 全 A 可交易股票池历史
4. 指数回测时，不再依赖单独的“指数完整行情库”，而是采用：
   - 全 A 主数据
   - 对应 universe 历史过滤
   - 历史时点成分口径回测

### 四、张量实验与选股主线

1. 在正式数据之上构建“股票 × 因子 × 时间”的张量实验管线。
2. 继续保留并对比以下模型路径：
   - `CP`
   - `Tucker`
   - `PCA`
3. 选股层需要支持按指定交易日输出候选股票列表，至少包含：
   - 股票代码
   - 综合评分
   - 所属市场
   - 所属股票池
   - 主要贡献因子
4. 正式配置文件至少维护：
   - `formal_hs300.yaml`
   - `formal_sz50.yaml`
   - `formal_zz500.yaml`
5. 各配置通过 universe 历史文件从全 A 主数据中过滤出对应股票池，而不是各自维护重复的完整主数据文件。

### 五、GPU 与高性能计算路线

1. 计算后端优先统一到 `PyTorch` 主路径。
2. 运行时需要支持：
   - `device=cpu`
   - `device=cuda`
   - `device=auto`
3. 在 CUDA 可用时，优先使用 GPU 路径执行主要数值计算。
4. 后续对热点算子预留：
   - `Triton`
   - 原生 `CUDA`
   的优化接口。
5. 首阶段不要求一开始就把所有算法全部改写成 `Triton/CUDA`，而是先通过 `PyTorch` 打通统一设备层，再根据 profiling 决定下沉哪些热点算子。

### 六、Web 端与系统形态

1. 后端继续采用 Python，并优先使用 `FastAPI`。
2. 前端继续采用 `React + Vite`。
3. Web 端需要支持：
   - 发起实验运行
   - 查看历史实验
   - 查看指标结果
   - 按指定日期浏览候选股
   - 在 `HS300`、`SZ50`、`ZZ500` 三类正式股票池之间切换
4. 输出目录应提供结构化产物，供 Web 端直接读取，例如：
   - `run_manifest.json`
   - `selection_candidates.json`
   - `factor_summary_*.json`
   - `factor_association_*.json`

### 七、与旧方案相比的关键修正

1. 不再把中证 A500 作为正式实验主目标。
2. 不再把“三指数并集公司”作为正式长期主存储方案。
3. 不再把正式全量数据和 smoke test 样例数据混在一起。
4. 不再使用“先按指数分别存多套完整行情/财务主数据”的思路，而是改为“全 A 主数据 + universe 历史过滤”。
5. 明确将正式全量数据窗口固定为 `2015-01-01` 到 `2026-04-01`。
6. 明确加入 `CSV -> Parquet` 的双形态落盘规划。
7. 明确把 `PyTorch + CUDA` 作为第一阶段 GPU 主路径，并为 `Triton/CUDA` 预留接口。
