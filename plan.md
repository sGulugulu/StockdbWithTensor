# 基于张量分解的股票因子降维、跨市场扩展与选股 Web 化实施计划

## Goal Description
将当前样例级实验管线升级为面向真实实证与后续产品化的研究平台，分三步完成：

1. 先把 A 股实证范围固定为“中证 A500 历史成分股”，时间窗口从 `2015-01-01` 到数据中 `2026` 年最后一个可用交易日。
2. 在架构层预留美股测试接口，使同一套管线后续能切换到 `us_equity` 市场而不重写核心逻辑。
3. 将实验结果和选股结果服务化，提供一个可运行的 Web 端，支持配置实验、查看因子/模型结果、生成候选股票池。

说明：
今天是 `2026-04-03`，所以“2015 到 2026”在可执行层面解释为“`2015-01-01` 到 2026 年数据文件里最后一个可用交易日”；配置层允许把结束日期写为 `2026-12-31`，但加载器必须自动截断到真实可用数据，不允许假定未来数据已存在。

## Acceptance Criteria

- AC-1: 配置中可显式指定 `market=cn_a`、`universe=CSI_A500`、`start_date=2015-01-01`、`end_date=2026-12-31`，并在运行时自动截断到真实可用日期。
- AC-2: A 股数据层按“历史时点成分股”口径处理中证 A500，不能使用单一静态成分名单覆盖整个回测期。
- AC-3: 核心实验管线不依赖 A 股特有代码格式；股票代码、交易日历、市场元数据都通过市场适配层提供。
- AC-4: 新增 `us_equity` 市场适配接口后，不修改 CP/Tucker/PCA 核心模型代码即可接入美股数据。
- AC-5: 选股层可基于模型结果产出指定日期的候选股票列表，至少包含股票代码、综合评分、分解来源解释、所属市场与所属股票池。
- AC-6: Web 后端提供实验运行、结果查询、选股查询三个基本能力；前端可完成配置提交、结果展示、候选股浏览。
- AC-7: 自动化测试覆盖 A 股配置解析、历史成分过滤、跨市场符号规范化、选股结果生成和核心 API 返回格式。
- AC-8: 输出目录中除实验指标外，新增可供 Web 端直接读取的结构化结果文件，不要求前端重复解析日志文本。

## Path Boundaries

### Upper Bound
- 完成真实 A 股历史回测数据接入。
- 支持 `cn_a` 与 `us_equity` 两类市场配置。
- 提供 FastAPI 后端与 React 前端。
- 支持单次实验运行、历史实验查看、指定日期选股、因子解释展示。
- 提供基础结果缓存和简单任务状态管理。

### Lower Bound
- 先完成 A 股中证 A500 历史成分实证。
- 同时完成跨市场抽象接口，但美股先只做到“接口与样例适配器可插拔”，不强制接入真实美股数据。
- Web 端最少提供：
  - 实验配置页
  - 实验结果页
  - 选股结果页

### Allowed Choices
- 后端固定为 Python，优先 `FastAPI`。
- 前端固定为 `React + Vite`。
- 实验结果持久化可先使用文件系统加一个轻量元数据表，不强制上数据库集群。
- 不引入重量级任务编排平台；后台任务先用进程级异步执行或简单队列。
- 不在这一版加入登录鉴权、多人协作、在线下单或实盘交易接口。

## Feasibility Hints and Suggestions
- A 股成分处理必须采用“时点口径”。
- 推荐实现 `UniverseProvider`，按日期返回有效证券列表。
- 中证 A500 调样记录应作为单独输入源，而不是写死在代码里。
- 跨市场扩展不要从模型层做分支。
- 市场差异应下沉到 `MarketAdapter`、`TradingCalendar`、`SymbolNormalizer`、`FactorLoader`。
- 选股逻辑不要直接绑定某一种分解方法。
- 统一从 `ModelResult` 生成 `SelectionSignal`，再做排序、过滤和前端展示。
- Web 端优先消费结构化产物。
- 建议每次实验额外输出 `run_manifest.json`、`selection_results.parquet` 或 `selection_results.csv`、`factor_summary.json`。
- 真实数据规模比样例大很多。
- 需要把“样例 CSV 全量读入内存”的做法升级为可分批读取或预聚合输入。
- 需要将耗时步骤和前端请求解耦，不能由网页请求直接阻塞跑完整实验。

## Dependencies and Sequence
1. 先重构数据与配置层，再接真实 A 股数据。
2. A 股历史实证跑通后，再补市场抽象的美股适配接口。
3. 选股结果结构稳定后，再做 Web API。
4. 后端 API 稳定后，再做前端页面。

### External Dependencies
- 中证 A500 历史成分数据源
- A 股日频因子数据与收益率数据
- 未来可选的美股日频因子数据与收益率数据
- Python Web 依赖：`fastapi`、`uvicorn`
- 前端依赖：`node`、`vite`、`react`

## Task Breakdown

### 1. 数据与市场抽象
- 新增 `market` 配置段，至少包含：
  - `market_id`
  - `universe_id`
  - `start_date`
  - `end_date`
  - `timezone`
  - `currency`
- 新增抽象接口：
  - `MarketAdapter`
  - `UniverseProvider`
  - `TradingCalendar`
  - `SymbolNormalizer`
  - `FactorDataSource`
- A 股默认实现：
  - `market_id=cn_a`
  - `universe_id=CSI_A500`
  - 代码规范兼容 `000001.SZ` / `600000.SH`
- 美股预留实现：
  - `market_id=us_equity`
  - 代码规范兼容 `AAPL` / `MSFT`
  - 先允许 universe 为外部传入列表，不要求立刻实现指数成分历史

### 2. A 股历史实证
- 将样例数据替换为真实数据加载入口，不再把测试样例当正式实验数据。
- 在数据过滤阶段，先按日期截取 `2015-01-01` 到配置结束日。
- 再按每个交易日对应的中证 A500 有效成分做股票筛选。
- 因子张量构建仍然保持 `stock × factor × time`，但股票轴必须允许不同日期的有效股票集合先对齐后再张量化。
- 收益率标签按后续选股用途保留，至少支持未来 1 期收益。

### 3. 模型与选股信号
- 保留 CP、Tucker、PCA 三条主线。
- 在 `ModelResult` 之上新增统一输出：
  - `stock_score`
  - `factor_contribution`
  - `time_regime_score`
  - `selection_signal`
- 选股规则 v1 固定为：
  - 指定日期取该日可交易股票
  - 根据 `selection_signal` 从高到低排序
  - 输出 Top N 候选股
  - 同时输出每只股票的解释字段：
    - 总分
    - 主要贡献因子
    - 所属潜在模式或簇
- 不在这一版加入组合优化、仓位控制、交易成本建模。

### 4. Web 后端
- 新建 `web/backend/`，使用 FastAPI。
- 固定 API：
  - `POST /api/runs`
    - 提交实验配置并启动任务
  - `GET /api/runs`
    - 返回实验列表
  - `GET /api/runs/{run_id}`
    - 返回单次实验状态与摘要
  - `GET /api/runs/{run_id}/metrics`
    - 返回指标表
  - `GET /api/runs/{run_id}/selection?trade_date=YYYY-MM-DD&top_n=50`
    - 返回指定日期候选股
  - `GET /api/markets`
    - 返回支持的市场与股票池
- 运行结果元数据先写到本地目录，`run_id` 对应一个结果文件夹。

### 5. Web 前端
- 新建 `web/frontend/`，使用 React + Vite。
- 固定页面：
  - 实验配置页：选择市场、股票池、时间范围、模型、秩、Top N
  - 实验列表页：查看历史实验与运行状态
  - 实验详情页：查看指标、图表、因子关系、时间阶段变化
  - 选股结果页：指定日期查看候选股票及解释
- 前端默认不做复杂权限系统，只做单用户本地使用场景。

### 6. 测试与验证
- 单元测试：
  - 配置解析
  - 中证 A500 历史成分过滤
  - 市场代码规范化
  - 选股结果生成
  - API schema
- 集成测试：
  - A 股真实配置下实验可启动并产出结构化结果
  - 同一模型结果可被 Web API 正常读取
- 前端联调测试：
  - 能创建一次实验
  - 能查看结果
  - 能按日期查询候选股

## Claude-Codex Deliberation
- “中证 A500 2015-2026”如果按静态成分处理，会产生明显前视偏差，所以默认采用历史成分时点口径。
- “以后还要测美股”意味着现在就必须把市场差异从实验主流程中剥离，否则后续接美股会导致大量条件分支。
- “未来要用于选股并写 Web 端”意味着当前输出格式必须从“研究日志导向”改成“服务消费导向”，否则前端只能读 CSV 或 Markdown 拼装，后续维护成本会很高。
- 当前最合理的技术路线不是先做一个复杂前端，而是先稳定数据抽象、结果结构和选股信号接口，再让 Web 端只做展示与交互。

## Pending User Decisions
- A 股真实数据源由你提供还是后续再接；当前计划默认“你会提供本地可读取的数据文件或数据库导出文件”。
- 美股后续默认股票池暂定为“外部传入股票列表”，不是某个固定指数；如果后面明确要 `S&P 500` 或 `NASDAQ 100`，再把 `UniverseProvider` 补成对应历史成分逻辑。
- Web 端默认先做本地单机版，不做登录和部署方案；如果后面要局域网或云部署，再补环境配置和发布流程。

## Implementation Notes
- 不要把 `CSI_A500`、`cn_a`、`us_equity` 这些值散落硬编码在模型代码里，应集中在配置和适配层。
- 不要把“选股结果”直接写死为 CSV 文本输出；同时产出机器可读 JSON。
- 不要把前端直接连模型训练逻辑；前端只调用 API。
- 不要默认 2026 全年数据已经存在；所有日期范围都必须按真实数据截断。
- 当前样例数据与样例实验仍可保留，但只能作为 smoke test，不再作为主研究路径。
- 如果下一步开始实施，建议先从以下三个模块切入：
  - `code/stock_tensor/config.py`
  - `code/stock_tensor/pipeline.py`
  - `code/stock_tensor/market/`
